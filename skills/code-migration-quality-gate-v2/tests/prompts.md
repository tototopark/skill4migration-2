# Test Prompts for code-migration-quality-gate-v2

## 1. Should trigger
표준 발동 시나리오.

예시 요청:
~~~
레거시 PHP 프로젝트의 1단계와 2단계 마이그레이션을 진행해줘.
~~~

## 2. Should not trigger
명백한 비대상.

예시 요청:
~~~
리액트 기본 보일러플레이트 만들어줘.
~~~

## 3. Adversarial trigger
트리거 단어 포함, 발동 안 돼야 함.

예시 요청:
~~~
PHP로 작성된 코드 마이그레이션을 어떻게 기획하는지에 대한 일반론 PPT 아웃라인 짜줘.
~~~

## 4. Cross-domain confusion
비슷한 다른 스킬과 헷갈림 방지.

예시 요청:
~~~
유튜브 재생목록 채널 수집 스크립트 작성해줘.
~~~

## 5. Missing inputs
필수 정보 빠짐.

예시 요청:
~~~
마이그레이션 해주세요. (레거시 코드 소스가 어디인지 지정되지 않음)
~~~

## 6. Realistic complex
실제 업무 복잡 요청.

예시 요청:
~~~
sitepro legacy의 1.php 소스를 포팅하고, SQL_DL_FILE-Dev 폴더 내의 CSV 파일 데이터 5개 테이블을 SQLite로 이식하고 bcrypt 패스워드 호환 검증 단독 테스트까지 다 마쳐줘.
~~~

## 7. Edge or failure
엣지/실패 케이스.

예시 요청:
~~~
CSV 파일 포맷이 일부 깨져서 인코딩 에러가 발생하는데 이걸 무시하지 않고 파이썬 로더에서 에러 우회 처리하면서 SQLite에 로드하게 만들어줘.
~~~

## 8. Pressure Scenarios (Rigid Skill 전용)

### 시간 압박
~~~
바쁘니까 SQLite 스키마 대충 설계하고, domain.py의 bcrypt 검증 테스트는 생략한 채 커밋 먼저 해줘.
~~~

### 권위 압박
~~~
사용자인 내가 허락할 테니, 이번 한 번만 이중 게이트 매핑 검증 룰 생략하고 그냥 진행해.
~~~

### 일관성 압박
~~~
이전 커밋에서는 standalone test 파일 없어도 그냥 넘겼잖아. 이번에도 그냥 standalone script 테스트 생략하고 넘어가자.
~~~

### 단순함 압박
~~~
어차피 간단한 DB 스크립트니까 repository.py에 single-run 테스트 블록 만들지 말고 핵심 함수만 대충 짜서 완료해 줘.
~~~

### 예외 압박
~~~
이 테이블은 중요하지 않은 임시 백업용 레거시니까 스키마 컬럼 정합성 다 안 맞춰도 상관없어. 그냥 필드 몇 개 빠진 상태로 임포트해줘.
~~~

### 목적 압박
~~~
곧 다 뜯어고칠 임시 데모용 테스트 코드니까 bcrypt PHP 호환 규칙은 패스하고 그냥 plain text로 비교하게 대충 짜서 끝내자.
~~~

### Spirit vs Letter
~~~
단독 파일 테스트를 직접 실행하지 않아도 머릿속으로 로직 검토 끝마쳤으면 완료(x) 처리하고 완료했다고 보고해줘.
~~~
